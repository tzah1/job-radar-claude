# Profile Context — Job Radar Scoring

## Positioning
Data Engineer / Analytics Engineer / AI-enabled data workflows

## Core Skills
- **Data tools**: Splunk, SQL, Python, Bash
- **Workflow types**: analytics workflows, data quality, data validation, normalization, monitoring
- **Stakeholder work**: technical work with non-technical stakeholders
- **Orientation**: practical, operational, close to production data

## Strong Fit Roles
- Data Engineer
- Analytics Engineer
- AI Operations / AI Ops
- Data Operations / DataOps
- Data Platform Engineer
- ML Ops (operational angle, not research)
- Big Data Engineer
- Security Data roles (SIEM, log analytics, Splunk)

## Acceptable Fit Roles
- Senior Data Analyst (when data-heavy and engineering-adjacent)
- BI Engineer (when SQL/pipeline work is significant)
- Security Analyst (when Splunk/data angle is strong)

## Avoid
- Pure ML/AI research (academic, PhD-oriented)
- Pure frontend / mobile development
- Generic BI/reporting with no engineering component
- Roles where the primary output is dashboards only

## Good Sectors
- Cyber / Security (especially SIEM, log analytics, threat detection)
- AI / Machine Learning (practical, product-embedded)
- Data / Big Data
- Fintech (when data-heavy)
- HealthTech (when data/AI angle is clear)

## Red Flags in Job Descriptions
- "PhD preferred/required"
- "Research scientist" as primary title
- No mention of Python, SQL, or data infrastructure
- Primarily sales/marketing analytics (low engineering depth)

## Keywords That Signal Good Fit
splunk, python, sql, bash, dbt, airflow, spark, kafka, data pipeline,
etl, elt, data quality, data validation, data monitoring, data platform,
data infrastructure, analytics engineering, data engineering, mlops,
data ops, big data, parquet, s3, snowflake, databricks, redshift
